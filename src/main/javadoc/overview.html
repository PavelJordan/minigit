MiniGit API documentation - javadoc

Introduction

MiniGit is a git clone coded in java with the main git functionality implemented. It also supports multiple minigit-repository handling (i.e., add/commit/tag/branch... are called on multiple repositories at the same time). This is a home program project for the Java programming course nprg013 at CUNI for the academic year 2025/26. It handles the main git commands like add, commit, checkout, merge, status log, and more, effectively implementing three-way merge and immutable git-like objects storage with blobs, trees, and commits.

Architecture

After loading the commands through ServiceLoader, the main class finds the appropriate command based on the first argument and executes it with the rest of the arguments. The command then uses the minigit structures to perform the necessary operations. You can see both commands and structures packages below.

Main packages

  • cz.cuni.mff.jordanpa.minigit.commands - CLI command implementations loaded through ServiceLoader
  • cz.cuni.mff.jordanpa.minigit.structures - repository structures with their business logic
  • cz.cuni.mff.jordanpa.minigit.misc - helper classes and algorithms, including diff and merge support, Author and FileHelper

Persistence

MiniGit stores immutable repository objects in the .minigit/objects directory and addresses them by SHA-1 hash (first two letters are subdirectory, the rest are the files themselves). Other types of objects are mutable references (branches, tags, HEAD, index), and those are saved alongside the "objects" directory. If you want to understand more, read the programmer documentation in the docs directory.
